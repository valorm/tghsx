Protocol Security

Our transparent approach to ensuring the safety and integrity of the tGHSX protocol and its users' funds.

Currently Unaudited The tGHSX smart contracts are in the MVP testnet phase and have not yet undergone a formal third-party security audit. Please use with caution and only with test funds on the Amoy network.

Internal Security Measures

While we work towards a formal audit, we have implemented a rigorous internal review process to minimize risks:

  • Comprehensive Test Suite: Our smart contracts have extensive unit and integration tests covering all core functions, including deposits, borrowing, liquidations, and withdrawals.
  • Peer Review Process: All contract code undergoes a mandatory peer review process by at least one other developer before being merged into the main branch.
  • Known Vulnerability Mitigation: We actively check against common smart contract vulnerabilities, such as reentrancy (mitigated with OpenZeppelin's ReentrancyGuard), integer overflows, and oracle manipulation.
  • Use of Battle-Tested Libraries: The protocol is built upon widely-used and audited libraries from OpenZeppelin and Chainlink to ensure a secure foundation.

Path to Mainnet Security

Our security roadmap is designed to be transparent and progressive, ensuring the protocol is robust before any mainnet deployment handling real assets.

  • Phase 1: Testnet Launch (In Progress)

    The protocol is currently live on the Polygon Amoy testnet to gather public feedback and identify potential issues in a controlled, no-risk environment.

  • Phase 2: Formal Audits (Planned)

    Before any mainnet launch, we will engage one or more reputable, independent security firms to conduct a full audit of the smart contract suite. All findings and subsequent fixes will be made public for full transparency.

  • Phase 3: Bug Bounty Program (Planned)

    Following the audits, we will launch a bug bounty program on a platform like Immunefi to incentivize continuous review of our code by the broader security community.

Responsible Disclosure Policy

Security is a collective effort. If you discover a potential security vulnerability in our smart contracts or web application, we ask that you report it to us privately so we can address it before it can be exploited.

Scope

This policy applies to the following assets currently on the Polygon Amoy Testnet:

  • The `CollateralVault.sol` smart contract.
  • The `TGHSXToken.sol` smart contract.
  • The tGHSX web application hosted at its official URL.

Out of scope: Any third-party services, denial-of-service (DoS) attacks, and social engineering attempts.

How to Report

Please send a detailed report of the vulnerability to our secure email address:

security@tghsx.com

Your report should include, if possible:

  • A clear description of the vulnerability and its potential impact.
  • Step-by-step instructions to reproduce the issue.
  • Any relevant proof-of-concept code, transaction hashes, or screenshots.

Our Commitment

  • We will provide a timely acknowledgement of your report (within 48 hours).
  • We will work diligently to validate, remediate, and release a fix for the vulnerability.
  • We will maintain open communication with you throughout the process.
  • For all valid, first-reported vulnerabilities, we will offer public acknowledgement and a reward, subject to the terms of our future bug bounty program.
